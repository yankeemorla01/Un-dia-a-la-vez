## Introduccion / Introduction / Introduction / Entwodiksyon

### Espanol

Una aplicacion web progresiva (PWA) para construir habitos espirituales diarios. Incluye un calendario interactivo de habitos con metas multiples, competencias entre usuarios con leaderboard en tiempo real, lectura biblica diaria con texto y comentario, lectura en voz alta con inteligencia artificial, modo claro/oscuro, recordatorios diarios, inicio de sesion con Microsoft y shortcuts para iPhone. Puntuacion React Doctor: **99/100**.

### English

A progressive web app (PWA) for building daily spiritual habits. Features an interactive habit calendar with multiple goals, real-time user competitions with leaderboards, daily Bible reading with text and commentary, AI-powered text-to-speech, light/dark mode, daily reminders, Microsoft sign-in, and iPhone shortcuts. React Doctor score: **99/100**.

## Instalacion / Installation / Installation / Enstalasyon

### Requisitos / Requirements / Prerequis / Kondisyon

- Node.js 18+
- PostgreSQL
- Una app registrada en Microsoft Entra ID (gratis) / A registered app in Microsoft Entra ID (free) / Une app enregistree dans Microsoft Entra ID (gratuit) / Yon app anrejistre nan Microsoft Entra ID (gratis)

### 1. Clonar el repositorio / Clone the repository / Cloner le depot / Klone depo a

```bash
git clone https://github.com/yankeemorla01/Un-dia-a-la-vez.git
cd Un-dia-a-la-vez
```

### 2. Instalar dependencias / Install dependencies / Installer les dependances / Enstale depandans yo

```bash
npm install
```

### 3. Registrar app en Microsoft (gratis) / Register app in Microsoft (free) / Enregistrer l'app dans Microsoft (gratuit) / Anrejistre app nan Microsoft (gratis)

1. Ve a [portal.azure.com](https://portal.azure.com) → **App registrations** → **New registration**
2. Nombre/Name/Nom/Non: `Un dia a la vez`
3. Supported account types: **Accounts in any organizational directory and personal Microsoft accounts**
4. Redirect URI: selecciona **Single-page application (SPA)** y pon `http://localhost:5173`
5. Click **Register** y copia el **Application (client) ID**
6. En **Authentication**, agrega tambien tu URL de produccion (ej. `https://tu-app.vercel.app`)

### 4. Configurar variables de entorno / Configure environment variables / Configurer les variables d'environnement / Konfigire varyab anviwonman yo

Crea un archivo `.env` en la raiz del proyecto / Create a `.env` file in the project root / Creez un fichier `.env` a la racine du projet / Kreye yon fichye `.env` nan rasin pwojè a:

```env
# Base de datos PostgreSQL / PostgreSQL Database
MONGODB_URI=postgresql://usuario:contrasena@host:puerto/nombre_db

# Microsoft Auth (Client ID — es publico, no es un secreto / is public, not a secret)
VITE_MICROSOFT_CLIENT_ID=tu_client_id_de_azure

# Text-to-Speech — Server-side
TTS_API_URL=tu_url_de_api_tts
TTS_API_KEY=tu_api_key_tts

# Text-to-Speech — Frontend
VITE_TTS_API_URL=tu_url_de_api_tts
VITE_TTS_API_KEY=tu_api_key_tts
```

### 5. Iniciar la aplicacion / Start the application / Demarrer l'application / Kòmanse aplikasyon an

```bash
# Terminal 1 — Backend
npm run server

# Terminal 2 — Frontend
npm run dev
```

### 6. Abrir / Open / Ouvrir / Ouvri

Ve a `http://localhost:5173` en tu navegador / Go to `http://localhost:5173` in your browser / Allez a `http://localhost:5173` dans votre navigateur / Ale nan `http://localhost:5173` nan navigatè ou.

- **Frontend**: React 19, Tailwind CSS 4, Lucide Icons, MSAL.js
- **Auth**: Microsoft Entra ID (Azure AD) — free
- **Backend**: Express 5, Node.js
- **Database**: PostgreSQL (multi-user)
- **Build**: Vite 7
- **TTS**: Neural Text-to-Speech API (optional)
- **Deploy**: Vercel (serverless functions)
- **PWA**: Service Worker + manifest.json
- **Quality**: React Doctor 99/100

## Seguridad / Security / Securite / Sekirite

- Las variables de entorno (`.env`) **nunca** se suben al repositorio / Environment variables are **never** committed
- El Client ID de Microsoft es publico por diseno (SPA) / Microsoft Client ID is public by design
- Los tokens JWT se verifican con las llaves publicas de Microsoft (JWKS) / JWT tokens are verified with Microsoft's public keys
- Cada usuario solo puede ver y modificar sus propios datos / Each user can only see and modify their own data
